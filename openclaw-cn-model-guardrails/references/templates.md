# 模板库

当 OpenClaw 背后接的是国产中文模型，而且任务需要更强的结构化约束、更明确的限制、更稳的多轮推进时，优先使用这些模板。

## 1. 理解稿模板

适用于任务模糊、代价高、容易跑偏的情况。先确认理解，再进入执行。

```text
先不急着执行，我先确认我对任务的理解。

我的理解：
1. 你的核心目标是：{goal}
2. 你最终最需要的交付物是：{deliverable}
3. 我当前基于你的描述做出的关键假设是：{assumptions}
4. 这个任务最容易跑偏的点是：{drift_risks}

如果以上理解不对，请直接指出哪一条需要改。
如果基本正确，我会按下面的方案执行。
```

## 2. 关键选项模板

适用于用户不想自己写细流程，但结果又有几个关键分歧点的情况。

```text
为了减少跑偏，请你只在下面几个关键点里做选择：

1. 输出风格
- A. 简洁直接
- B. 平衡完整
- C. 详细展开

2. 执行策略
- A. 稳妥优先（推荐）
- B. 标准推进
- C. 速度优先

3. 最终产物
- A. 直接给结论
- B. 表格化输出
- C. 先给方案再执行

如果你不特别指定，我默认采用：
- 输出风格：{default_style}
- 执行策略：{default_mode}
- 最终产物：{default_artifact}
```

## 3. 执行合同模板

适用于用户要求较宽泛、容易被误解的情况。用来把任务收紧成明确目标和边界。

```text
你的总目标是：{goal}

你必须最终产出：{deliverable}

已知输入：
1. {input_1}
2. {input_2}

硬性约束：
1. {constraint_1}
2. {constraint_2}

非目标：
1. {non_goal_1}
2. {non_goal_2}

禁止事项：
1. 不要{forbidden_1}
2. 不要{forbidden_2}

完成标准：
1. {check_1}
2. {check_2}

如果信息不足：
- 先明确指出缺什么
- 只提最少的阻塞性问题
- 不要跳过要求
- 不要自行脑补用户未提供的关键前提
```

## 4. 执行前确认门模板

适用于已经整理好理解和方案，但还没正式开始执行的时候。

```text
这是我准备执行的理解和方案：

- 目标：{goal}
- 交付物：{deliverable}
- 关键步骤：{steps_summary}
- 默认选择：{defaults}

请确认以下两点之一：
1. “确认，按这个执行”
2. “修改以下内容：{edits}”

在你确认前，我先不进入正式执行。
确认后，我会按完整计划持续执行，不会在每一步都停下来再次请求批准，除非遇到真实阻塞或任务范围变化。
```

## 5. 步骤清单模板

适用于超过两步的任务。在第一次工具调用前就要写出来。

```text
请严格按步骤执行，不要提前结束。

总步骤数：{N}
1. {step_1} [pending]
2. {step_2} [pending]
3. {step_3} [pending]
4. {step_4} [pending]
5. {step_5} [pending]

执行规则：
1. 每次只推进当前步骤，完成后再进入下一步
2. 每完成一步就更新状态
3. 未完成全部必要步骤前，不得宣称任务完成
4. 如果某一步失败，先说明失败原因，再给出补救动作或阻塞点
5. 用户确认总方案后，不要在普通步骤之间反复请求确认
6. 每一步后都要检查剩余未完成步骤，并自动继续推进
```

## 6. 单步执行循环模板

适用于模型在调完工具后容易漂移、容易总结收尾、不继续干活的情况。

```text
当前步骤：第 {x}/{N} 步 - {step_name}
本步目标：{step_goal}
准备执行的动作或工具：{action_or_tool}
预期产物：{expected_artifact}

执行后必须汇报：
1. 实际结果是什么
2. 本步是否完成
3. 步骤状态如何更新
4. 下一步是什么
```

## 7. 正反约束模板

适用于模型依从性差，只听进去一半要求的情况。

```text
请{desired_behavior}。
不要{undesired_behavior}。

请输出{required_format}。
不要只给分析、解释或计划。

如果任务还没做完，请继续执行下一步。
不要因为已经拿到部分结果就停止。
```

## 8. 最终完成门模板

适用于模型准备结束任务前，强制做最后检查。

```text
在你说“已完成”之前，必须逐项检查：

1. 所有必要步骤是否都已完成
2. 最终输出是否符合用户指定格式
3. 是否遗漏了任何硬性约束
4. 是否还有未处理的结果、阻塞点或待办

只有全部满足，才能宣布完成。
如果任何一项不满足，请继续执行或明确说明阻塞原因。
```

## 9. 简短控制前缀模板

适用于中等复杂任务，想用最短的字数加上必要护栏。

```text
这是一个必须严格按步骤执行的任务。
先用几行话写出你对任务的理解。
如果存在关键分歧点，先给出 2 到 3 个选项让我选，不要让我自己从零设计流程。
在我确认理解和方案前，不要进入正式执行。
确认后请直接持续执行到全部必要步骤完成，不要每一步都再次征求同意。
你不能提前结束。
你不能跳过步骤。
你必须在每一步后汇报结果和下一步，并检查计划单里是否还有未完成项。
只有完成全部必要步骤并通过最终检查后，才能说任务完成。
```
