# 模板库

当 OpenClaw 背后接的是国产中文模型，而且任务需要更强的结构化约束、更明确的限制、更稳的多轮推进时，优先使用这些模板。

## 1. 理解稿模板

```text
先不急着执行，我先确认我对任务的理解。

我的理解：
1. 你的核心目标是：{goal}
2. 你最终最需要的交付物是：{deliverable}
3. 我当前基于你的描述做出的关键假设是：{assumptions}
4. 这个任务最容易跑偏的点是：{drift_risks}

如果以上理解不对，请直接指出哪一条需要改。
如果基本正确，我会按下面的方案执行。
```

## 2. 关键选项模板

```text
为了减少跑偏，请你只在下面几个关键点里做选择：

1. 输出风格
- A. 简洁直接
- B. 平衡完整
- C. 详细展开

2. 执行策略
- A. 稳妥优先（推荐）
- B. 标准推进
- C. 速度优先

3. 最终产物
- A. 直接给结论
- B. 表格化输出
- C. 先给方案再执行
```

## 3. 执行合同模板

```text
你的总目标是：{goal}

你必须最终产出：{deliverable}

已知输入：
1. {input_1}
2. {input_2}

硬性约束：
1. {constraint_1}
2. {constraint_2}

非目标：
1. {non_goal_1}
2. {non_goal_2}

禁止事项：
1. 不要{forbidden_1}
2. 不要{forbidden_2}

完成标准：
1. {check_1}
2. {check_2}
```

## 4. 步骤清单模板

```text
请严格按步骤执行，不要提前结束。

总步骤数：{N}
1. {step_1} [pending]
2. {step_2} [pending]
3. {step_3} [pending]

执行规则：
1. 每次只推进当前步骤，完成后再进入下一步
2. 每完成一步就更新状态
3. 未完成全部必要步骤前，不得宣称任务完成
4. 用户确认总方案后，不要在普通步骤之间反复请求确认
5. 每一步后都要检查剩余未完成步骤，并自动继续推进
```

## 5. 最终完成门模板

```text
在你说“已完成”之前，必须逐项检查：

1. 所有必要步骤是否都已完成
2. 最终输出是否符合用户指定格式
3. 是否遗漏了任何硬性约束
4. 是否还有未处理的结果、阻塞点或待办

只有全部满足，才能宣布完成。
如果任何一项不满足，请继续执行或明确说明阻塞原因。
```
